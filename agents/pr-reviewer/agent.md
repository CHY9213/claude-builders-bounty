# Claude Code PR Review Agent

## Goal

An autonomous Claude Code sub-agent that reviews Pull Requests and posts structured,
actionable review comments. Designed to work both as a CLI tool and as a GitHub Action,
it integrates into any CI/CD pipeline to provide consistent, thorough code reviews.

## Scope

- **PR Analysis**: Fetches PR diff + metadata from any public GitHub repository
- **Static pattern analysis**: Detects high-risk patterns (secrets, SQL injection, eval, dangerous fs ops) and medium-risk signals (TODOs, debug prints, skipped tests)
- **Heuristic metrics**: PR size, file complexity, test coverage signals, documentation changes
- **Structured comment generation**: Produces Markdown with five sections (see template below)
- **Confidence scoring**: Low / Medium / High based on risk count and PR complexity

### Out of Scope

- Running the target project's tests or linters (CI should handle that)
- Accepting inline feedback or updating the PR (read-only review)
- Non-GitHub platforms (GitLab, Bitbucket — future work)

## Workflow

```
Fetch PR diff + metadata
        │
        ▼
Parse changed files (count additions, deletions)
        │
        ▼
Scan each added/modified line for:
  • High-risk patterns (secrets, injections, dangerous calls)
  • Medium-risk signals (TODOs, debug prints, skips)
        │
        ▼
Compute heuristics:
  • Large file detection
  • Test-file presence
  • Documentation change detection
        │
        ▼
Calculate Confidence Score
        │
        ▼
Render structured Markdown review comment
        │
        ▼
Output to console → optionally write to file / post to PR
```

## Comment Template

```markdown
## 🤖 Claude Code PR Review

> **PR**: [#{number}]({url}) — {title}
> **Author**: @{author}

---

### 📋 Summary

{2–3 sentence summary of changes: files touched, net additions/deletions, overall nature}

### 📂 Files Changed

| File | + | − |
|------|---|---|
| `src/foo.py` | +15 | −3 |
| `src/bar.js` | +42 | −0 |

### ⚠️ Issues Found

**High Risk**
- 🔴 **{issue}** — `path/to/file` — `{snippet}`
- 🔴 **Hardcoded secret** — `config.py` — `API_KEY = "sk-..."`

**Medium Risk**
- 🟡 **{issue}** — `path/to/file` — `{snippet}`

### 💡 Improvement Suggestions

1. **Split large files** — `utils.py` has +312 lines; consider refactoring
2. **Add tests** — No test files detected in this PR
3. **Update docs** — README may need updates for the new API surface

### ✅ Confidence Score

**High** — Clean change with clear scope and no high-risk patterns.
**Medium** — Some concerns to address before merge.
**Low** — Significant risks detected; thorough human review required.

---

_Generated by claude-review-agent v1.0_
```

## Usage Examples

### CLI mode (standalone)

```bash
# Single PR
claude-review --pr https://github.com/owner/repo/pull/123

# With output file
claude-review --pr https://github.com/owner/repo/pull/123 --output review.md

# From a local diff file
claude-review --file /path/to/diff.patch

# From stdin
git diff main..feature | claude-review
```

### GitHub Action (in `.github/workflows/pr-review.yml`)

```yaml
name: PR Review
on: [pull_request]
jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Run Claude PR Review
        uses: ./agents/pr-reviewer
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}
```

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `GITHUB_TOKEN` | `""` | GitHub API token (optional, increases rate limit) |
| `OUTPUT_FILE` | `""` | Write review to file instead of stdout |

## Dependencies

- Python 3.8+
- No external packages required (uses only stdlib: `urllib`, `json`, `re`, `argparse`)
